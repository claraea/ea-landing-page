Enterprise Architecture Repository

EA/CMDB — Enterprise Architecture com profundidade de negócio única.

EA/CMDB é o EA Repository nativo do framework Clara/EA. Único no mercado a integrar hierarquia completa de negócio — Mercado → Nicho → Marca → Veículo Legal → Produto → Canal — com grafo nativo de alta performance, descoberta automática de infraestrutura e geração de IaC via LLM. Posiciona-se como alternativa enterprise-grade ao LeanIX, Ardoq e Alfabet, com âncora open-source.

O que é

EA Repository, não CMDB operacional.

CMDBs operacionais — ServiceNow, BMC, Device42 — nasceram para ITSM: incident, change, inventário técnico. Servem ao help desk, ao NOC, ao change manager. Respondem "este servidor está funcionando?".

EA Repositories — LeanIX, Ardoq, Alfabet, Mega HOPEX — nasceram para arquitetura empresarial e decisões estratégicas. Servem ao arquiteto enterprise, ao CTO, ao CIO. Respondem "este produto de negócio depende de quais capacidades?".

EA/CMDB pertence ao segundo segmento — mas vai além: é um EA Repository com hierarquia de negócio mais profunda que qualquer concorrente, com discovery automático de infraestrutura que nenhum EA tool tradicional faz, e com geração de IaC via LLM que nenhum competidor sequer planeja.

Capacidades

Onde o EA/CMDB é imbatível

Seis capacidades que combinadas inexistem em qualquer EA Repository do mercado. Nenhum concorrente entrega esse conjunto — e várias delas, individualmente, são únicas no segmento global.

I

Hierarquia Clara/EA — 12 camadas

Mercado → Segmento → Nicho → Marca → Veículo Legal → Produto → Canal → Modalidade, conectados como entidades de primeira classe em grafo. Ningúem desce a esse nível: LeanIX e Alfabet param em Business Capability. Únicos a modelar o próprio negócio em toda a sua hierarquia de valor.

II

Grafo nativo (ArangoDB)

Banco de dados em grafo nativo — não emulado via foreign keys. 29 tipos de aresta tipados, named graphs múltiplos, traversal multi-hop em AQL. Apenas Ardoq usa grafo nativo entre os concorrentes — e sem nossa profundidade de modelo de negócio.

III

Discovery automático de IaC

Parser nativo de Terraform HCL, ingestão de manifests Kubernetes, sync com Entra ID/LDAP, ingestão de custos AWS/GCP. Para times que vivem em IaC, elimina a desatualização crônica dos EA tools — o repositório se atualiza sozinho. Inexistente em LeanIX, Alfabet, Ardoq.

IV

IaC Generation via LLM

Único EA Repository do mercado que executa: gera Terraform e Ansible a partir do inventário modelado. O arquiteto desenha a plataforma → o sistema produz IaC pronto para revisão e apply. Fim do ciclo "documentação → implementação manual". Sem paralelo global.

V

FinOps end-to-end

Custo de cloud distribuído pela hierarquia Clara/EA — única plataforma capaz de responder "qual o custo total de infra AWS para sustentar o nicho de Cartão de Crédito PF da marca Visa?" via traversal de grafo. Concorrentes param em custo por aplicação ou por capability.

VI

IPAM, Skills e Org Snapshots

VPC, subnet, alocação de IP e cabeamento físico no mesmo grafo da arquitetura de negócio. Skills por funcionário ligadas ao organograma e ao inventário. Snapshots temporais da estrutura organizacional. Capacidades que EA Repositories tradicionais consideram fora de escopo — e que aqui são primeira classe.

Arquitetura

Três camadas, um único grafo executável

1. Camada de modelo

Implementa a hierarquia Clara/EA inteira como tipos de entidade e relacionamento — Mercado, Nicho, Marca, Veículo Legal, Produto, Canal, Capability, Application, Database, Cloud Resource, IP, Pessoa, Skill. ArangoDB como grafo nativo, AQL para queries.

2. Camada de descoberta

Workers (Temporal) ingerem continuamente: parser de Terraform HCL, discovery de Kubernetes, sync de Entra ID/LDAP, ingestão de billing AWS/GCP, sync com ITSM (GLPI, ServiceNow, Jira). REST API para alimentação programática. Plugin SDK para conectores customizados.

3. Camada de uso

APIs de consulta, MFEs modulares (Topology, Observability, FinOps, Org Chart), TIME Matrix, Technology Radar, dashboards executivos. Worker-iac gera Terraform/Ansible via LLM. Webhooks inbound/outbound, multi-tenancy nativo, i18n PT-BR / EN / ES.

Posicionamento

Onde competimos — e onde não.

Onde ganhamos

  • → Empresas que praticam ou querem praticar o framework Clara/EA
  • → Times que vivem em IaC (Terraform, Kubernetes) e rejeitam EA tools desatualizados
  • → Mercado brasileiro: financeiro, fintechs, bancos digitais, seguradoras, telecom
  • → Quem precisa de profundidade de negócio além do Application Portfolio padrão
  • → Quem quer EA enterprise sem pagar US$ 100k–500k de licença anual

Onde não competimos (ainda)

  • → Empresas que exigem ArchiMate/TOGAF como linguagem formal obrigatória
  • → Quem precisa de BPMN integrado ao EA
  • → Setores altamente regulados que exigem GRC formal (SOX, COBIT, BACEN mapping)
  • → Contratos enterprise com SLA 24/7 e suporte premium contratual

Gaps reconhecidos. Roadmap competitivo público — interoperabilidade ArchiMate, surveys colaborativos e target-state diff em desenvolvimento.
